# Frame Interpolation for Smooth Video Output

## Problem

Videos generated by playwright-recast from Playwright's `recordVideo` WebM output appear choppy. The source WebM has a variable, often low frame rate from browser recording. Currently no frame interpolation exists ‚Äî speed adjustments use only ffmpeg's `setpts` (PTS manipulation), and the output FPS flag (`-r`) duplicates frames rather than generating intermediate ones.

## Solution

Add a new `.interpolate()` pipeline stage that uses ffmpeg's `minterpolate` filter to generate motion-compensated intermediate frames, achieving smooth 30-60fps output.

## API

### Fluent Pipeline

```typescript
await Recast
  .from('./traces')
  .parse()
  .speedUp({ duringIdle: 3.0 })
  .subtitlesFromSrt('./narration.srt')
  .autoZoom()
  .interpolate({
    fps: 60,              // Target FPS (default: 60)
    mode: 'mci',          // 'dup' | 'blend' | 'mci' (default: 'mci')
    quality: 'balanced',  // 'fast' | 'balanced' | 'quality' (default: 'balanced')
  })
  .render({ format: 'mp4' })
  .toFile('demo.mp4')
```

### CLI

```bash
npx playwright-recast -i ./traces --interpolate
npx playwright-recast -i ./traces --interpolate --interpolate-fps 30
npx playwright-recast -i ./traces --interpolate --interpolate-mode blend --interpolate-quality fast
```

## InterpolateConfig

```typescript
export type InterpolateMode = 'dup' | 'blend' | 'mci'
export type InterpolateQuality = 'fast' | 'balanced' | 'quality'

export interface InterpolateConfig {
  fps?: number                    // Target FPS (default: 60)
  mode?: InterpolateMode          // minterpolate mi_mode (default: 'mci')
  quality?: InterpolateQuality    // Quality preset (default: 'balanced')
}
```

### Modes

| Mode | Description |
|------|-------------|
| `dup` | Duplicate frames ‚Äî fastest, no artifacts, no smoothing |
| `blend` | Linear blend between frames ‚Äî fast, soft crossfade effect |
| `mci` | Motion-compensated interpolation ‚Äî best quality, generates true intermediate frames |

### Quality Presets

Quality presets map to ffmpeg minterpolate parameters (only relevant when `mode: 'mci'`):

| Preset | `mc_mode` | `me_mode` | `vsbmc` | `search_param` | Use case |
|--------|-----------|-----------|---------|----------------|----------|
| `fast` | `obmc` | `bilat` | `0` | `32` | Quick preview, CI |
| `balanced` | `aobmc` | `bidir` | `1` | `64` | Default, good quality/speed ratio |
| `quality` | `aobmc` | `bidir` | `1` | `400` | Final production video |

## Pipeline Position

Interpolation runs as Phase 3.75 ‚Äî after all visual effects (cursor, click, zoom), before audio processing and final encode:

1. Phase 2: Speed processing (`setpts`)
2. Phase 3: Cursor overlay
3. Phase 3.25: Click effects
4. Phase 3.5: Zoom (`zoompan`)
5. **Phase 3.75: Frame interpolation (`minterpolate`)** ‚Äî NEW
6. Phase 3.7: Click sound
7. Phase 4: Audio padding
8. Phase 5: Final encode

Rationale: visual effects are applied at the source frame rate (cheaper, no interpolation artifacts in overlays), then interpolation generates smooth intermediate frames from the fully composited video.

### ffmpeg Filter

```
minterpolate=fps=60:mi_mode=mci:mc_mode=aobmc:me_mode=bidir:vsbmc=1:search_param=64
```

Single ffmpeg call per video. No chunking or parallelization in v1 ‚Äî complexity not justified for typical demo video lengths (30-120s).

## Constraints

- **ffmpeg-only** ‚Äî no external AI tools, Docker, or GPU requirements
- **Opt-in** ‚Äî disabled by default; user must explicitly call `.interpolate()`
- **Single-threaded** ‚Äî minterpolate is single-threaded in ffmpeg; processing time scales with video duration and quality preset
- **Performance expectation** ‚Äî `balanced` quality on 60s 1080p video: ~2-5 minutes processing
